- Setup and enable RPC server in nosonode software listening on port 8078 (default)

- Setup a proxy to address CORS issues experienced with browsers.
	If using apache for the proxy, perform the following apache steps:
		Uncomment the following modules in apache.conf
			#LoadModule proxy_module modules/mod_proxy.so
			#LoadModule proxy_http_module modules/mod_proxy_http.so
			#LoadModule headers_module modules/mod_headers.so
		Add the following directives to apache config between the<Directory></Directory> tags:
			Header set Access-Control-Allow-Methods "GET, POST, PUT, DELETE, OPTIONS"
			Header set Access-Control-Allow-Headers "Authorization, Content-Type"
		Add the following to the apache virtual hosts file:
			##---CORS proxy for RPC
			Listen 8079
			<VirtualHost *:8079>
			    ServerName server.com:8079
			    ServerAlias server.com
			    <Location />
			        ProxyPass http://serverIP:8078/
			        ProxyPassReverse http://serverIP:8078/
			        Header set Access-Control-Allow-Origin "*"
			    </Location>
			     SSLEngine on
			    SSLCertificateFile "C:/servercert.pem"
			</VirtualHost>
